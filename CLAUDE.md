# Crossplane Provider MongoDB Atlas

Upjet-based Crossplane provider for MongoDB Atlas. Generates CRDs from the `mongodb/terraform-provider-mongodbatlas`.

## Project Structure

- `config/` — Provider configuration (external names, resource overrides, per-resource configs)
  - `config/external_name.go` — Central external name mappings for all resources
  - `config/provider.go` — Provider setup, default resource options, resource configurator registration
  - `config/cluster/` — Cluster-scoped resource configs (one subdir per resource group)
  - `config/namespaced/` — Namespaced-scoped resource configs (mirrors cluster/)
  - `config/cluster/common/common.go` and `config/namespaced/common/common.go` — Shared helpers (version constants, ID extractors). These MUST remain separate because they embed scope-specific package paths (`APISPackagePath`, `ExtractResourceIDFuncPath`).
- `apis/cluster/` and `apis/namespaced/` — Generated API types (do not edit manually)
- `internal/controller/` — Generated controllers (do not edit manually)
- `package/crds/` — Generated CRDs (do not edit manually)
- `examples-generated/` — Generated example manifests (do not edit manually)
- `cluster/images/provider-mongodbatlas/Dockerfile` — Provider container image

## Key Commands

- `make generate` — Regenerate APIs, controllers, CRDs, and examples from config
- `make build` — Build provider binaries
- `make local-deploy` — Build and deploy provider locally (requires a running cluster)
- `make e2e` — Run end-to-end tests
- `go build ./config/...` — Quick compile check for config changes

## Code Generation (upjet)

All resources are code-generated by upjet. The generation pipeline:

1. **External name config** (`config/external_name.go`) defines which TF resources to include and how their IDs map to Crossplane external names
2. **Resource configurators** (`config/cluster/*/config.go`) set per-resource overrides: `ShortGroup`, `Kind`, `Version`, `References`, `LateInitializer`, custom `GetIDFn`/`GetExternalNameFn`
3. **Default resource options** (`config/provider.go`) apply to all resources: ShortGroup reset and external name configuration
4. `make generate` runs upjet to produce `apis/`, `internal/controller/`, `package/crds/`, and `examples-generated/`

### Important upjet behaviors

- `TemplatedStringAsIdentifier("", tmpl)` parses `{{ .parameters.X }}` from the template and adds those parameters to `IdentifierFields`. Parameters in `IdentifierFields` are **removed from the generated CRD schema** (no `forProvider` field, no Ref/Selector). Our `templatedStringAsIdentifier` wrapper clears `IdentifierFields` to prevent this.
- `DisableNameInitializer` is checked at code generation time and baked into `zz_controller.go`. You must run `make generate` after changing it.
- When changing a resource's `Version` (e.g., v1alpha2 → v1alpha3), delete the old `apis/*/GROUP/OLD_VERSION/` directory before running `make generate`, or stale files will cause build errors.
- Two provider scopes exist (cluster and namespaced) that generate into separate API packages. Config files under `config/cluster/` and `config/namespaced/` are near-mirrors of each other.

## External Name Patterns

In `config/external_name.go`, the `externalNameConfigs` map uses these constructor functions:

- `identifierFromProvider()` — Wraps `config.IdentifierFromProvider`. Provider-assigned ID; all parameters stay in the CRD schema.
- `templated(tmpl, externalNameField)` — Wraps `templatedStringAsIdentifier`. Template-based ID (e.g. `"{{ .parameters.project_id }}/{{ .parameters.name }}"`). Clears `IdentifierFields` so template parameters remain in the CRD schema with Ref/Selector support. The `externalNameField` (a TF state field name) is used to set the `crossplane.io/external-name` annotation after the first successful observe.
- `{ExternalName: config.ParameterAsIdentifier("field")}` — Uses a parameter as external name. **Removes** that parameter from the CRD schema. Only used for `mongodbatlas_project`.

### External name annotation initialisation

Each `externalNameEntry` carries an `initCRDField` (the camelCase `forProvider` field name). Before the first observe, `ExternalNameInitializers()` registers an initializer (via `initFromField`) that seeds `crossplane.io/external-name` from that field instead of `metadata.name`. This is required so Observe-only management policies work correctly — upjet skips observe when the annotation is absent.

### How `templatedStringAsIdentifier` differs from upstream

`config.TemplatedStringAsIdentifier("", tmpl)` would normally add template parameters to `IdentifierFields`, removing them from the CRD schema. Our wrapper:
1. Passes an empty `nameField` to avoid tying the external name to `metadata.name`.
2. Sets `DisableNameInitializer = false` so the name initializer runs (required for Observe-only policies).
3. Clears `IdentifierFields` so parameters like `project_id` stay in `forProvider` with Ref/Selector support.
4. Sets a custom `GetExternalNameFn` that reads `externalNameField` from the TF state.

## Conventions

- Every resource must have an `AddResourceConfigurator` in its config.go with explicit `r.References` for `project_id` (or other cross-resource refs)
- Resources in the root API group have `ShortGroup = ""` (handled by `defaultResourceOptions` in provider.go)
- Resources in sub-groups set `r.ShortGroup = "groupname"` explicitly in their configurator
- Version constants (`VersionV1Alpha2`, `VersionV1Alpha3`) live in `config/cluster/common/common.go` and `config/namespaced/common/common.go`
- The Dockerfile sets `NO_COLOR=1` and `TF_CLI_ARGS_import=-no-color` to suppress ANSI in terraform output
